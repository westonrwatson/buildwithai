Build Lab 01

Ship your first game.

Learn to prompt, build a Wordle clone, add your own flair, and deploy it live on Vercel.

Talk to the AI

How to prompt.

Part 01
Role
Tell the AI who it is. Give it a persona that fits the task.
"You are an expert web developer."
Part 02
Task
Tell it exactly what to build or do. Be direct and specific.
"Build me a Wordle clone."
Part 03
Context
Give it the details, constraints, and requirements it needs.
"Single index.html. No frameworks."

Prompting tips.

01
Be specific
Vague prompts get vague results. Describe the layout, colors, behavior, and constraints you want. The more detail you give, the closer the first output lands.
02
Give constraints
Tell it what you don't want. "No frameworks. Vanilla JS only." Constraints help the AI produce cleaner, more focused code.
03
Iterate
Your first prompt is a draft. Ask it to change one thing at a time — small edits compound fast.
04
Ask why
Don't just copy the code. Ask: "Explain what this block does." You'll understand it faster than reading docs.
05
Paste the error
If something breaks, copy the error message and paste it directly into the chat. The AI can debug its own code.
06
Describe the vibe
You don't need technical terms. Say "Make it feel more editorial and clean" or "It should feel like a luxury brand."
The build

Wordle clone.

Phase 01
Generate it.
Paste the starter prompt into Cursor. Let the AI build a complete, playable Wordle clone in a single index.html file. Test it locally before moving on.
Phase 02
Make it yours.
Pick at least two customizations below and prompt the AI to apply them. Iterate one change at a time until it feels like your version.
Phase 03
Ship it.
Push to GitHub, connect to Vercel, and get a live URL. Save the link — you'll submit it after Lab 02.
Starter prompt — copy & paste this first
You are an expert web developer. Build a fully playable Wordle clone as a single index.html file. Include: 6 guesses per game, 5-letter words only, a built-in word list of at least 50 common words, an on-screen keyboard, tile flip animations, color feedback (green = correct spot, yellow = wrong spot, gray = not in word), win and lose states with a Play Again button, and a clean modern UI. Vanilla HTML, CSS, and JavaScript only — no frameworks, no external libraries. Make it responsive and work on mobile.
Phase 02

Add your flair.

Custom theme
"Switch to a dark theme with [YOUR COLOR] as the accent. Update the keyboard and tiles to match."
Rename it
"Change the title to [YOUR NAME]'s Wordle and add a short tagline under it."
Tile animations
"Add a satisfying flip animation when each row of tiles is revealed."
Win streak stats
"Add a stats panel showing games played, win %, and current streak. Persist stats in localStorage."
Hard mode
"Add a hard mode toggle — revealed hints must be used in subsequent guesses."
Custom word list
"Replace the word list with themed words — sports, food, or inside jokes with friends."
Phase 03

Push & deploy.

01 — Push to GitHub

In Cursor, ask the AI: "Help me push this project to a new GitHub repo." Or use Source Control → Initialize Repository → Publish to GitHub. Name it my-wordle.

02 — Deploy on Vercel

Go to vercel.com → Add New Project → Import your GitHub repo → Deploy. Copy your live URL when it's done.
